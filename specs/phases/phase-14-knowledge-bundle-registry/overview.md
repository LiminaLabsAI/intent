# Phase 14 — Knowledge Bundle Registry & Cyclic Refinement

> **Governing decision:** [ADR-0003 — Knowledge Bundle Registry](../../decisions/0003-knowledge-bundle-registry.md) (the versioned deliverable model + naming). Operationalizes [ADR-0002](../../decisions/0002-working-memory-build-flow-and-cost.md) ("OKF markdown files, full spec compliance, versioned").
> **Roadmap move:** inserts as Phase 14; the existing "Precedent & Memory" renumbers to 15 (it consumes this registry); Governance 15→16; Human Review 16→17; Handoff 17→18; Normalization 18→19; Multi-source 19→20; Progressive Autonomy 20→21; V2 21→22; Execution Agent 22→23.

## Goal

Graduate the built deliverable from a destructive scratch field (`record.files`, overwritten on every Rebuild) into a **first-class, versioned, immutable-on-publish, addressable Knowledge Bundle** — Google's Open Knowledge Format's own unit — and ship the **conversational cyclic-refinement loop** that produces its versions. The **Intent Registry** (design doc §4.6, image-aligned) keeps holding Intents; a built Intent now publishes **versioned OKF Knowledge Bundles**. This is lifecycle stage 7 maturing (registry for *outcomes*, not just *intentions*) and the load-bearing seam between the *clarity* half (Phases 7–13) and the *control* half (Governance, Phase 16) — both renumbered Precedent (15) and Governance (16) read what this phase writes.

## Key decisions (→ ADR-0003)

| Decision | Choice |
|---|---|
| Versioned subject | **The OKF Knowledge Bundle** (a directory of concept `.md` files); not the intent, not memory, not "artifact". Draft UI word = "artifact". |
| Registry scope | **Intent Registry stays the system-of-record holding Intents**; bundles are *versioned outputs of an intent*, namespaced `INT-0007/vN`. No separate Knowledge Registry. |
| History shape | **Two-tier** — draft checkpoints (git-commit-like, append-only, free restore) + published versions (git-tag-like, immutable forever, citable). Nothing is ever mutated or deleted. |
| Refinement engine | **Hybrid** — full re-build per refine by default (correct-by-construction, no stale-file bugs), PLUS a manual **"regenerate just this concept"** targeted path. Auto-delta deferred to V2 (needs a Rule-11-locked evaluator; identical-regen rate is the prerequisite data this phase starts producing). |
| Refine loop surface | **Continuous** — same chat continues; any post-build message is a refine instruction (agent perceive detects "bundle exists + delta message" → refine move). No dedicated mode, no inline file editing (both deferred). |
| Version label | **Auto-label = the user's refine-request text**, not a timestamp; user may name on Publish. |
| Publish gate (V1) | **Ungated** — the user Publishes when satisfied; governance coupling deferred to Phase 16 (reuses this state machine: request-changes → draft → vN+1). |
| Identity | **INT-namespaced**: shareable `/i/INT-0007/v1`. Global `KB-XXXX` id deferred to when bundles leave the intent. |
| What's in a version | OKF concept files + **snapshot of the Understanding it was built from** (provenance for Phase 15 precedent) + cost actuals + persona + label + parent pointer. |
| Bundled changelog | `log.md` autogenerated inside every published bundle (OKF-native), so a bundle is self-describing of its own history even outside Flow. |

## BundleVersion state machine

```
  (refine / build)         Publish            next Publish
  ┌────────────┐       ┌────────────┐      (auto)        ┌────────────┐
  │   DRAFT    │──────▶│  PUBLISHED │──────────────────▶│ SUPERSEDED │  (still
  │  #k …#k+1  │       │   = vN     │                    │            │   readable,
  └─────┬──────┘       └─────┬──────┘                    └─────┬──────┘   citable)
        │                    │ (user Deprecate)                 │
        │ Restore-as-new-    ▼                                  │ Archive
        │ draft from any  ┌────────────┐  (user Archive)       ▼
        │ version       ─▶│ DEPRECATED │──────────────────▶ ┌─────────┐
        └──────────────    └─────┬──────┘                    │ ARCHIVED│ (admin-
                              │ Publish again → vN+1         └─────────┘  only,
                              └─▶ PUBLISHED(vN+1) supersedes…              retrievable)
```

**Invariants:** append-only (no mutation, no deletion); exactly one `latestPublished` pointer per bundle (auto-supersession on next Publish); published content survives supersession/deprecate/archive unchanged; restore is forward (a fresh DRAFT seeded from any version's content — history untouched).

## Scope

### In (V1)
- Registry schema (`KnowledgeBundle` / `BundleVersion` / `ConceptFile`), event types, OKF-validator, migration of existing `record.files` → seed DRAFT.
- Refine engine: continuous refine (full re-build) + per-concept regen; drafts as immutable checkpoints with "what changed" summary; content-hash deltas; version label = refine-request text.
- Publish / supersede / deprecate / archive / restore-as-draft + shareable `/i/INT-0007/vN`.
- Studio UI: Drafts & Versions drawer, per-concept diff vs parent, restore, Publish (with version naming), per-file Regenerate, download bundle (`.zip` + per-file).
- Minimal discovery: reuse the sidebar intent history; surface a "published" badge + latest version count. `log.md` autogenerated inside bundles.

### Out (→ roadmap / backlog)
- V2 attached-system execution; global `KB-XXXX` id / cross-intent sharing / external share token / external git-repo distribution.
- Approval-gated Publish (Phase 16 Governance — reuses this state machine).
- ENH-003 mode-scaled build depth — deferred to a `/hotfix` quick-task (it also touches the build prompt; fold there to keep this phase focused).
- Named branches / forks (Figma-style).
- Phase 15 (renumbered) Precedent & Memory reads this registry; not built here.

## Acceptance criteria

1. A refine after a Build **appends a DRAFT** (does not overwrite `record.files`); the drawer lists it labelled by the user's change-request text; per-concept diff vs the prior draft shows only touched files.
2. `Publish` creates an immutable, addressable `/i/INT-0007/v1`; a second publish auto-supersedes v1; v1 remains readable/citable forever.
3. Restoring any version → a new DRAFT seeded from it → refine → publish as vN+1, **without mutating history**.
4. Deprecate / Archive flag a version without deleting it; immutable content survives.
5. Any message after a built artifact is perceived as a refine instruction and runs a refine build; per-file Regenerate runs the targeted path and produces a draft differing only in that concept.
6. Published bundles are **OKF-conformant** (frontmatter + `type` on every concept, valid `index.md`/`log.md`); `log.md` reflects refine-history.
7. Full suite green (`npm test`, currently 82) + `tsc` 0.

## Non-goals

No V2 execution engine, no global bundle id / cross-intent sharing / external share token / external git export, no named branches/forks, no approval-gated publish, no inline bundle editing, no precedent memory, no ENH-003.
